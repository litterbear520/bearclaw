bearclaw 架构全景

从 mini_agent.py 单文件出发，逐步拆解为模块化架构。每个模块只做减法——与 nanobot 保持一致的接口和命名，去掉尚未学到的复杂性。

6
AgentRunner.run() 多轮 LLM 对话循环：provider.chat() → 解析 tool_calls → ToolRegistry.run() → 追加结果
7
SessionManager.save() 持久化会话 → Dream.run() 消费压缩摘要，用受限工具编辑记忆文件
8
提取最后一条 assistant 消息 → OutboundMessage → bus.publish_outbound() → 显示给用户

Agent 核心

agent/
loop.py
CLASS AgentLoop 主循环，协调所有子系统

run() — 消费 inbound → 压缩 → 构建上下文 → runner → _save_turn → dream → 发布 outbound
_save_turn(session, messages, skip) — 跳过初始消息，将新增消息打时间戳后存入 session
_run_dream() — 构建 dream prompt → 复用 runner 执行（ephemeral，不存 session）→ 推进 cursor

CLASS ContextBuilder 上下文构建器

_load_bootstrap_files() — 加载 AGENTS.md, SOUL.md, USER.md

CLASS OpenAICompatProvider (LLMProvider) OpenAI 兼容 API 适配

_parse() — OpenAI 响应 → 统一 LLMResponse
factory.py
FUNC make_provider() 根据 LLM_BACKEND 环境变量创建对应 provider 实例

CLASS EditFileTool (Tool)
create(ctx) — 覆写：从 ctx.workspace 构造

CLASS GlobTool (Tool)
create(ctx) — 覆写：从 ctx.workspace 构造
