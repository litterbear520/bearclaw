Core Data Flow

消息如何流转

消息通过一个异步 MessageBus 流转，它将聊天渠道与 Agent 核心完全解耦。整个流程分为四个阶段。

1

渠道接收

各平台的 Channel 实现监听外部消息（Telegram、Discord、微信等），将原始消息封装为统一的 InboundMessage 事件，发布到 MessageBus。

nanobot/channels/base.py
2

Agent 循环协调

AgentLoop 消费入站消息，管理 session key、加载上下文（系统提示词、会话历史、Skills），然后将完整的上下文交给 AgentRunner。

nanobot/agent/loop.py
3

LLM 对话与工具执行

AgentRunner 执行多轮 LLM 对话循环：发送消息给 Provider、接收 tool call 请求、执行工具、将结果反馈给 LLM，直到模型给出最终回复。

nanobot/agent/runner.py
4

响应分发

最终回复封装为 OutboundMessage，通过 MessageBus 路由回对应的 Channel，由 Channel 负责格式化和发送给用户。

nanobot/bus/queue.py

Subsystems

六大核心子系统

每个子系统各司其职，通过接口解耦协作。了解这六个模块就理解了 nanobot 的骨架。

⟲
Agent 核心

整个系统的处理引擎。AgentLoop 管理会话、Hook、上下文构建；AgentRunner 执行 LLM 多轮对话和工具调用。支持并发工具执行和流式输出。

agent/loop.py · runner.py
⬡
LLM Providers

统一的 LLM 提供商接口。所有 Provider 继承 LLMProvider 基类，实现 send_message 方法。通过 Factory + Registry 模式按模型名自动路由。

providers/base.py · factory.py
⇌
Channels

平台集成层。每个 Channel 继承 BaseChannel，将平台消息转换为 InboundMessage。通过 pkgutil 扫描和 entry-point 插件自动发现。

channels/base.py · manager.py
⚙
Tools

暴露给 LLM 的能力集合：文件系统操作、Shell 执行（带沙盒）、Web 搜索/抓取、MCP Server、定时任务、子Agent 生成、图像生成等。同样通过插件机制自动发现。

agent/tools/registry.py
◉
Memory 与 Session

会话历史持久化，采用 Dream 两阶段记忆整合。Session Manager 管理每会话历史、上下文压缩（当上下文过长时自动 compact）、TTL 过期清理。

agent/memory.py · session/
◧
WebUI

基于 Vite + React 的单页应用。通过 WebSocket 多路复用协议与 Gateway 通信。Dev Server 自动代理 /api、/auth 和 WebSocket 请求。

webui/src/App.tsx

# AgentRunner 的核心循环（简化版）

class AgentRunner:
    async def run(self, spec: AgentRunSpec) -> LLMResponse:
        messages = spec.initial_messages

        for iteration in range(spec.max_iterations):
            # 1. 发送消息给 LLM Provider
            response = await spec.runtime.provider.send_message(messages)

            # 2. 检查是否有 tool call 请求
            if not response.tool_calls:
                return response  # 最终回复，结束循环

            # 3. 执行所有请求的工具
            for call in response.tool_calls:
                result = await spec.tools.execute(call.name, call.arguments)
                messages.append(tool_result_message(call.id, result))

            # 4. 将工具结果反馈给 LLM，继续循环

Getting Started

三种使用方式

无论是直接 CLI 启动、通过 Python SDK 嵌入、还是使用 WebUI 交互，nanobot 都提供一致的 Agent 体验。

CLI 命令行

通过 nanobot gateway 启动网关服务，加载配置并初始化所有已启用的 Channel。

$ nanobot gateway
Python SDK

通过 nanobot.py 以编程方式创建 Agent 实例，适合嵌入到其他应用中。

from nanobot import Nanobot
WebUI

Vite + React SPA，通过 WebSocket 与 Gateway 通信。支持聊天、设置、文件预览、代码高亮。

$ cd webui && bun run dev
